# 视觉模型与多模态能力规划 : 0%

> 创建时间：2026年5月13日  
> 范围：图片输入识别、AI 返回图片显示与保存、文生图、视频附件识别、视频保存、视频生成 API。  
> 排序原则：先整理当前已完成能力，再按工作量从小到大规划下一步。

## 一、当前已完成能力

### 1. 模型多模态能力标记已具备
- [√] 已定义模型输入能力：`Text`、`Image`、`Audio`、`Video`、`File`。
- [√] 已定义模型输出能力：`Text`、`Image`、`Audio`、`Video`。
- [√] 模型设置页已支持配置输入/输出能力标记。

### 2. 图片附件输入链路已具备
- [√] UI 已支持选择图片附件。
- [√] UI 已支持拖放图片附件。
- [√] 发送消息时会识别图片 MIME 类型。
- [√] 图片会复制到工作区资源目录。
- [√] 图片会写入 `ChatMessageAssets` 资源索引表。
- [√] 图片会通过 `DataContent` 加入 AI 消息内容，具备发送给视觉模型识别的主链路。

### 3. 图片资源保存与历史恢复已具备
- [√] 用户图片附件可保存到资源目录。
- [√] 图片资源索引可按会话和消息加载。
- [√] 历史恢复时可将图片资源重新构造成 `DataContent` 回灌给 AI 上下文。

### 4. 图片显示已有基础能力
- [√] Markdown 渲染器支持本地路径图片。
- [√] Markdown 渲染器支持 HTTP/HTTPS 图片。
- [√] Markdown 渲染器支持 data URI 图片。
- [√] 用户图片附件发送时会追加 Markdown 图片链接，具备基础显示能力。
- [×] AI 返回图片的自动展示链路仍需完善：需要把 AI 生成/返回的图片资源稳定转换为可见图片卡片或内联图片。

### 5. 音频/视频资源展示已有基础能力
- [√] 已支持识别音频 MIME 类型。
- [√] 已支持识别视频 MIME 类型。
- [√] 已有非图片资源卡片展示控件。
- [√] 音频和视频可通过资源卡片调用系统默认外部软件播放。
- [√] 当前阶段不需要内置音频/视频播放器，外部软件播放即可。

### 6. AI 返回多媒体资源保存已有通用基础
- [√] AI 响应中的 `DataContent` 可保存到资源目录。
- [√] 保存逻辑可按 MIME 类型区分 `images`、`audio`、`video`、`files`。
- [√] 可写入 `ChatMessageAssets`，标记为 `generated` 资源。
- [×] 视频生成后的保存与 UI 关联流程还需要完善和验证。

## 六、阶段验收标准

- 图片附件能发送给支持视觉输入的模型并得到有效识别结果。
- AI 返回图片能立即显示，并能在重启应用后从历史中恢复显示。
- 文生图接口可生成图片，生成结果能保存、索引、展示。
- 视频附件能发送给支持视频输入的模型，且不支持时有明确提示。
- AI 返回或生成的视频能保存为资源，并以卡片形式展示和外部打开。
- 视频生成 API 能完成任务创建、状态跟踪、结果保存和失败处理。
