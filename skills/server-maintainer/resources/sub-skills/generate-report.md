# Generate Monitor Report（生成监控报表）

## 功能描述
连接所有已注册的服务器，采集 CPU、内存、硬盘、网络等监控数据，生成 Markdown 格式报表并保存到指定位置。

## 调用触发
当用户请求：
- "生成监控报表"
- "生成服务器报表"
- "检查服务器状态"
- "看看服务器监控情况"
- 其他类似意图

**主技能将调用此子技能执行报表生成任务。**

## 保存规范

### 保存位置
- 路径：`{工作目录}\Reports` 文件夹
- 自动创建：如果 Reports 文件夹不存在，自动创建

### 文件格式
- 格式：Markdown (`.md`)
- 编码：UTF-8

### 命名规则
- 格式：`YYYYMMDD.md`
- 示例：`20260410.md`
- 说明：纯数字日期，不包含横杠、下划线等符号
- 覆盖策略：同一天多次生成报表时，覆盖当日已有文件

---

## 执行流程

1. 读取 Servers 文件夹，获取所有服务器列表
2. 检查 Reports 文件夹是否存在，不存在则创建
3. 循环连接每台服务器：
   - 读取 server.md 获取连接信息
   - 建立 SSH 连接
   - 执行监控命令采集数据
   - 记录连接状态和指标数据
4. 生成 Markdown 报表内容
5. 保存到 `{工作目录}\Reports\YYYYMMDD.md`
6. 返回文件完整路径给用户

## 参数

| 参数名 | 类型 | 必需 | 说明 |
|--------|------|------|------|
| serverList | Array | 否 | 服务器列表，不传则默认所有服务器 |
| thresholds | Object | 否 | 阈值配置，不传则使用默认值 |

## 输出

- **控制台**: 实时输出连接状态和采集进度
- **文件**: Markdown 报表保存到 Reports 文件夹
- **返回值**: 报表文件完整路径

## 异常处理

| 异常类型 | 处理方式 |
|----------|----------|
| 服务器连接失败 | 标记为离线，继续处理下一台 |
| 部分指标采集失败 | 记录为"数据不可用"，继续其他指标 |
| Reports 文件夹创建失败 | 报错并终止，提示用户手动创建 |
| 文件写入失败 | 报错并返回失败原因 |
